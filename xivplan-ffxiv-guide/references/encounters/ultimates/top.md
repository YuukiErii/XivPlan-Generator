# TOP Mechanic Cards

## Scope

Use these cards for The Omega Protocol (Ultimate), commonly abbreviated `TOP`. TOP strategies are sensitive to priority systems and data-center conventions; confirm the user's macro before producing exact assignments.

## Pantokrator

- 副本：TOP / The Omega Protocol (Ultimate)
- 阶段：Beetle Omega
- 常见别名：Pantokrator、全能之主、Panto、转圈放圈
- 机制类别：`bait`、`rotation`、`sequence`
- 原理：玩家围绕场地按稳定节奏移动，在多轮诱导与个人职责中保持间距，避免路径相撞。
- 默认解法：固定起点、旋转方向和每轮移动距离；把每一拍的诱导点与下一拍等待点分开。
- 常见优化：让全员使用相同旋转规则，减少临场判断；近战在合法时贴目标圈旋转。
- 常见失误：移动过快或过慢导致 bait 重叠；箭头没有标拍数。
- 可类比机制：转圈放圈、地火引导、clockwise bait、Exaflare follow。
- 可迁移部分：统一旋转、节拍、间距检查、每拍拆图。
- 不应照搬：TOP 的具体 Debuff、诱导次数和旋转半径要按新机制确认。
- XivPlan 图示拆分：起点；前半旋转；后半旋转；复位。
- 需要确认的信息：旋转方向、诱导对象、每拍间隔、圈半径、是否有角色差异。
- 参考来源：[Icy Veins TOP full guide](https://www.icy-veins.com/ffxiv/the-omega-protocol-ultimate-guides)
- 置信度：`medium`

## Party Synergy

- 副本：TOP / The Omega Protocol (Ultimate)
- 阶段：Omega-M and Omega-F
- 常见别名：Party Synergy、PS、男女组合技、Mid Glitch、Remote Glitch
- 机制类别：`tether`、`debuff`、`clone-memory`、`spread`
- 原理：同符号 pair 和近远距离条件共同决定站位，玩家还要观察 Omega-M/Omega-F 分身动作并进入正确安全区。
- 默认解法：先识别 pair 和 Mid/Remote，再观察分身 safe pattern；用 pair lane 解距离，用单独一步解分身 AoE。
- 常见优化：把符号 pair 映射为固定左右顺序，减少自由排序；图上单独画距离圈。
- 常见失误：只看符号不看 Mid/Remote；pair 站位正确但躲错分身攻击。
- 可类比机制：PlayStation pair、近远线、观察分身动作、形状 pair spread。
- 可迁移部分：符号 pair、距离条件、观察与执行分图。
- 不应照搬：Omega 分身动作、safe pattern 和具体 lineup 不能直接复制。
- XivPlan 图示拆分：识别符号与 Glitch；pair lineup；分身安全区；最终散开。
- 需要确认的信息：pair 标识、近远阈值、分身动作、最终散开规则。
- 参考来源：[Icy Veins TOP full guide](https://www.icy-veins.com/ffxiv/the-omega-protocol-ultimate-guides)
- 置信度：`medium`

## Hello World

- 副本：TOP / The Omega Protocol (Ultimate)
- 阶段：Omega Reconfigured
- 常见别名：Hello World、HW、世界状态、补丁
- 机制类别：`debuff`、`pass`、`bait`、`sequence`
- 原理：多种世界状态、Rot、Patch 和 AoE 职责按时间轴连续传递与消除，核心是状态分类、合法接触与固定 relay。
- 默认解法：先做状态表和时间轴，再为每个 bucket 画等待位、传递位、bait 位和回收位。
- 常见优化：用固定 relay 路线和明确的 pass 时机，避免玩家追逐；将 callout 限制为当前 beat。
- 常见失误：不同状态错误接触；图只画最终位置，不画 pass 的先后。
- 可类比机制：O12S `Hello World`、TEA Nisi、传毒、长时间 Debuff 舞蹈。
- 可迁移部分：状态 bucket、固定 relay、beat-by-beat 时间轴、接触禁区。
- 不应照搬：TOP 的世界状态类型、Patch 关系和具体 relay 顺序不能用于一般传毒机制。
- XivPlan 图示拆分：Debuff 表；第一次 relay；第一次 bait；第二次 relay；最终清理。
- 需要确认的信息：状态类型、持续时间、接触规则、bait 规则、清除条件。
- 参考来源：[Icy Veins TOP full guide](https://www.icy-veins.com/ffxiv/the-omega-protocol-ultimate-guides)
- 置信度：`medium`

## Run Dynamis Delta Sigma Omega

- 副本：TOP / The Omega Protocol (Ultimate)
- 阶段：Run: Dynamis
- 常见别名：Delta、Sigma、Omega、Dynamis、三运
- 机制类别：`sequence`、`tether`、`debuff`、`bait`、`tower`
- 原理：连续 trio 将早期出现的符号 pair、近远条件、世界状态和诱导职责重新组合；每段都要求先分类再进入固定 lane。
- 默认解法：Delta、Sigma、Omega 分开建卡与图；每段先画分类表，再画 bait、tower、line 或 final spread。
- 常见优化：尽量复用 `Party Synergy` 的 pair lineup；只在必须变化时重排。
- 常见失误：把三个 trio 当成一张超大图；延用上一段职责却没有写重置点。
- 可类比机制：Ultimate trio、大运动会、旧机制 remix、连续状态桶。
- 可迁移部分：复用早期 pattern、每段重置、分类后执行。
- 不应照搬：Delta、Sigma、Omega 的具体职责、塔位、Arm Unit bait 和世界状态顺序必须逐段确认。
- XivPlan 图示拆分：每个 trio 独立 4-8 步；段首分类；段尾 reset。
- 需要确认的信息：当前 trio、优先级体系、pair lineup、bait/tower 分配、世界状态顺序。
- 参考来源：[Icy Veins TOP Sigma guide](https://www.icy-veins.com/ffxiv/the-omega-protocol-ultimate-phase-5-guide-sigma)、[Icy Veins TOP Omega guide](https://www.icy-veins.com/ffxiv/the-omega-protocol-ultimate-phase-5-guide-omega)
- 置信度：`medium`
